【添加公告概述】

 【版本2.2、2.3】{添加公告弹窗；表单包括：公告类型（必填）}、标题（必填）、时间（必填）、正文内容（必填，不能超过 5 千字）}、上传文件（非必填，只能上传pdf文件且不得超过10M）、推送对象（必填，选择上报账号的基础上，还可以选择对应上报账号的机构职能、机构类别，机构职能、机构类别允许多选，选择后消息通知会推送给符合对应条件的上报机构）；

表单字段拆解：

0、【版本2.2】{公告类型 - 必填 - 单选，版本更新和操作手册只有在系统管理维护公告时才显示，消息通知则是在所有管理机构维护公告时都显示；}


1、消息标题 - 必填  - 文字输入框- 不能超过100个字


2、时间 - 必填  - 时间控件 - 精确到天，默认当天，不可选择未来时间


3、消息内容 - 必填  - 文本域输入框 -  不能超过5000字，右下角会显示  目前字数/5000


4、上传文件 - 非必填 - 只能上传pdf文件且不能超过10M，允许上传多个文件；上传文件后显示文件名如下：








5、推送对象 - 必填  - 选择项：管理账号和上报账号 - 可多选；当选择上报机构时，可以选择机构职能和机构类别，可以多选，选择后在对应符合条件的上报机构中会查看到对应的消息通知；

符合条件逻辑：

1、机构职能或者机构类别可以选择多个选项，只要上报机构中机构职能或者机构类别选择了一个对应的选项，那么就算这个选项符合条件，不允许出现机构职能或者机构类别选择空的同时又选择某一个选项的情况，但是允许只选择空或者选择多个非空选项【版本2.7】{下拉框会显示所有选中的内容，原型图画的只是大概示意仅供参考}

2、机构职能、机构类别需要同时符合要求，如果机构职能、机构类别都为空，那就表示选择所有上报机构，如果机构职能、机构类别只有一个为空，那么就是符合另一个条件的所有上报机构，如果机构职能、机构类别都不为空，那么就是同时符合两个条件的上报机构；
